# Intro to Debugging

## Description
This project focuses on the systematic process of finding and fixing logic errors in C programs using step-by-step execution tracing. By utilizing visualizers like Python Tutor, the goal is to observe control flow, inspect variable states over time, and eliminate common bugs such as off-by-one errors and incorrect variable updates without blindly rewriting code.

## Learning Objectives
* Understand what debugging is and why step-by-step tracing is essential.
* Execute C code line by line and inspect variable values at each step.
* Identify and fix common logic errors involving loop conditions, control variables, and function return values.

## Requirements
* Language: C (compiled with `gcc -Wall -Wextra -Werror -pedantic -std=gnu89`)
* No pointers, arrays, structs, dynamic memory, or file I/O allowed.
* Output must match exactly, including spaces and newlines.
